# 私人健身助手 - 减脂助手 设计规格

## 概述

移动端Web应用（手机浏览器访问），私人健身助手，以减脂为首要目标。纯前端、纯本地存储（localStorage）。

## 技术栈

- Vue 3 + Vite + TypeScript
- Vant 4 移动端UI组件库
- Tabbar页面切换，无路由库
- localStorage全量持久化

## 页面结构

Vant Tabbar底部4个Tab：

| Tab | 名称 | 内容 |
|-----|------|------|
| 首页 | 今日概览 | 当日热量缺口仪表盘、三餐摘要、运动摘要 |
| 饮食 | 饮食管理 | 当日三餐详情、食物列表、替换食物、调整克数 |
| 运动 | 运动管理 | 有氧/无氧运动计划、消耗进度 |
| 我的 | 个人中心 | 身体数据、目标设置、食物库管理 |

## 数据模型

```
UserProfile {
  性别, 年龄, 身高(cm), 体重(kg),
  颈围(cm), 腰围(cm), 臀围(cm),
  手动体脂率(可选), 目标体脂率,
  活动水平(5档: 久坐/轻度/中度/活跃/高强度)
}

DailyPlan {
  日期, 生成时间,
  目标摄入(千卡), 实际摄入,
  运动目标消耗(千卡),
  三餐: { 早餐: [{食物id, 克数}], 午餐: [...], 晚餐: [...] },
  运动: { 有氧: {类型, 时长(分钟), 消耗(千卡)}, 无氧: {类型, 时长, 消耗} }
}

FoodItem {
  id, 名称, 分类, 每百克: {热量(千卡), 蛋白质(g), 碳水(g), 脂肪(g)}
}
```

## 核心计算公式

1. **体脂率** — 美国海军法(颈围/腰围/臀围+身高)，手动输入可覆盖估算值
2. **BMR** — Mifflin-St Jeor公式
3. **TDEE** — BMR × 活动系数(1.2 / 1.375 / 1.55 / 1.725 / 1.9)
4. **每日摄入目标** — TDEE − 缺口(默认500千卡，可调)
5. **三餐分配** — 早30% / 午40% / 晚30%（如需加餐从晚餐额度中拆分）
6. **营养配比** — 蛋白质2g/kg体重，脂肪0.8g/kg，剩余热量分配碳水
7. **运动消耗** — 有氧60% / 无氧40%(可调)

## 食物数据库

预制约50种常见减脂食物，含每百克热量、蛋白质、碳水、脂肪数据。分类包括：
主食类(麦片、全麦面包、糙米饭、杂粮粥、红薯、玉米等)、蛋白质类(鸡胸肉、鸡蛋、牛肉、鱼肉、虾仁、豆腐等)、乳制品豆类(酸奶、牛奶、豆浆等)、蔬菜类(西兰花、菠菜、番茄、黄瓜、白菜等)、水果类(苹果、香蕉、蓝莓等)、坚果油脂类。

## 食谱生成 — 基于中国居民膳食指南

食谱结构遵循《中国居民膳食指南(2022)》核心原则，确保营养均衡、符合中国人饮食习惯：

### 膳食指南约束

| 原则 | 具体约束 |
|------|----------|
| 谷物为主 | 每日谷薯类250-400g，分配到三餐主食 |
| 蔬果充足 | 蔬菜300-500g/天，水果200-350g/天 |
| 奶豆补充 | 奶及奶制品约300g/天，大豆及坚果25-35g/天 |
| 适量动物性食物 | 鱼禽蛋瘦肉合计120-200g/天 |
| 少油少盐 | 烹调油<30g/天，盐<5g/天（不计入热量，仅提示） |

### 三餐模板结构

**早餐（总热量30%）**
- 主食1份：杂粮粥/全麦馒头/玉米/麦片/红薯 等
- 蛋白质1份：鸡蛋/豆浆/牛奶
- 蔬果或小菜1份

**午餐（总热量40%）**
- 主食1份：糙米饭/杂粮饭/全麦面条
- 荤菜1份：鸡胸肉/牛肉/鱼肉/虾仁/豆腐
- 素菜2份：至少2种蔬菜（深色蔬菜优先）

**晚餐（总热量30%）**
- 主食半份：少量粗粮或薯类
- 蛋白质半份：鱼类/豆制品（偏清淡易消化）
- 蔬菜1-2份

### 生成算法

1. 根据每日摄入目标，按三餐比例分配热量
2. 每餐按蛋白质2g/kg体重优先锁定蛋白质食物克数
3. 按膳食指南蔬果推荐量锁定蔬菜水果克数
4. 剩余热量分配给主食，确保总热量不超目标
5. 同类食物可随机轮换，避免连续重复
6. 优先选择低GI主食和高饱腹感食物

三餐各有默认模板，根据当日热量目标自动生成并缩放克数。用户可替换食物、增减克数。

## 交互流程

### 首次使用
1. 进入"我的"Tab → 填写身体数据 + 目标体脂率 + 活动水平
2. 保存后立即生成首日饮食和运动计划

### 日常使用
1. 每天首次打开App → 检测今日是否已有计划 → 无则自动根据最新身体数据生成
2. 首页展示当日热量缺口仪表盘、三餐摘要、运动摘要
3. 饮食Tab查看/调整具体食物和克数
4. 运动Tab查看/调整运动类型和时长
5. 我的Tab查看/更新身体数据、目标设置

### 热量缺口仪表盘
- 环形进度：实际摄入 / 目标摄入
- 运动消耗进度条
- 净热量 = 摄入 − TDEE − 运动消耗
- 缺口目标达成状态(绿/黄/红)

## 本地存储

- `fitness-profile` — UserProfile
- `fitness-plans` — DailyPlan[] (按日期索引)
- `fitness-food-db` — FoodItem[] (首次写入预设数据，用户可增改)
- `fitness-settings` — { 热量缺口, 三餐比例, 有氧无氧比例 }

## 非功能需求

- 响应式设计，适配320px-428px手机屏幕
- 首次加载 < 3秒
- 离线可用(可选PWA)
- 无任何网络请求，纯离线应用
